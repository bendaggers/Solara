# SOLARA Machine Learning Trading System (Version 3)

## Overview

This system automates trading decisions using a machine learning model to identify entry opportunities. The workflow consists of three main stages: data collection, model inference, and trade execution.

## Workflow

### Stage 1: Data Collection (MT5)
- **Tool**: `DataExporter.ex5` (MetaTrader 5 Expert Advisor)
- **Action**: Exports current market data to a CSV file
- **Output Location**:
`C:\Users\<your username>\AppData\Roaming\MetaQuotes\Terminal\<terminal id>\MQL5\Files`
- **Content**: CSV file containing OHLC data, technical indicators, and other features for multiple currency pairs

### Stage 2: Model Inference (Python)
- **Process**:<br>
&emsp;1. Load the CSV file generated by MT5<br>
&emsp;2. Preprocess data by removing unnecessary columns<br>
&emsp;3. Format data for model consumption<br>
&emsp;4. Pass data through ML model to identify entry signals<br>
- **Model**: Trained machine learning model that evaluates each candle for entry potential
- **Output**: List of currency pairs flagged as good entry opportunities

### Stage 3: Trade Execution
- **Backup Output**: Generate JSON file with qualified pairs for record-keeping
- **Execution**: Python script sends trade requests to MT5 terminal based on model recommendations
- **Integration**: Uses MT5's API or bridge to execute trades automatically


## System Components

### 1. Data Exporter (MT5 EA)
- Exports real-time market data to CSV format
- Configurable symbols, timeframes, and indicators
- Runs on timer or market event 

### 2. Python Processing Pipeline
- **Data Loader**: Reads CSV from MT5 output folder
- **Preprocessor**: Cleans and formats data for model input
- **Model Runtime**: Executes trained ML model on prepared data
- **Trade Qualifier**: Filters and validates model signals
- **JSON Exporter**: Creates backup record of qualified pairs
- **Trade Executor**: Sends orders to MT5 via API
- **Trade Logger**: Logs trade in json

### 3. Machine Learning Model
- Accepts formatted market data as input
- Outputs entry/no-entry classification per currency pair
- Can be retrained periodically with new data

## Running the System
1. Start MT5 and activate DataExporter.ex5
2. Run Python pipeline: **`python solara.py`**
3. Monitor outputs in the JSON file and MT5 terminal

## Output Files
**`raw_data.csv`**: Market data from MT5 (input)<br>
**`qualified_pairs.json`**: Model-selected pairs with timestamps (backup)<br>
**`execution_log.txt`**: Trade execution records (optional)<br>

**1. Install Python 3.11.7**

**2. Install Required Libraries**

Open your terminal or command prompt and run:<br>
`pip install numpy==1.26.4 pandas==2.2.1 scikit-learn==1.5.2 matplotlib==3.9.2 pandas-ta==0.3.14b0 MetaTrader5==5.0.4200`

These are the minimum packages required to run the scripts. Additional optional packages for extended functionality (like plotting, backtesting, or MetaTrader5 integration) can be installed as needed.

**3. Verify Installation**

You can verify that all required packages are installed correctly by running:

`python -c "import numpy, pandas, sklearn; print('All core packages installed successfully!')"`

**4. Notes**

This setup does not require a virtual environment. Packages will be installed system-wide.

Using the exact Python version (3.11.7) ensures compatibility with pre-trained models and avoids pickle issues.

For reproducibility or deployment to other machines, keep the requirements.txt file updated.
